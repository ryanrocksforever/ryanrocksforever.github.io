How to Implement Universal Design

Practical techniques for building accessible websites.

Semantic HTML: The Foundation

Using the right HTML elements is the most important thing you can do for accessibility. Screen readers and other assistive technologies rely on semantic structure.

Use Proper Headings

Headings create a document outline. Use them in order (h1, then h2, then h3) without skipping levels.

<!-- Good -->
<h1>Page Title</h1>
<h2>Section</h2>
<h3>Subsection</h3>

<!-- Bad: Skipping heading levels -->
<h1>Page Title</h1>
<h4>Why did we skip to h4?</h4>

Landmark Elements

Use HTML5 landmark elements to define page regions:

<header>Site header and logo</header>
<nav>Navigation links</nav>
<main>Primary content</main>
<aside>Sidebar content</aside>
<footer>Copyright and links</footer>

Lists for Lists

Use <ul>, <ol>, and <li> for listsâ€”not divs with bullet characters.

Buttons vs Links

  • <button>: For actions (submit, toggle, open modal)
  • <a>: For navigation (go to another page or location)

Images and Media

Alt Text for Images

Every <img> needs an alt attribute. The content depends on context:

<!-- Informative image -->
<img src="chart.png" alt="Sales increased 25% from Q1 to Q2">

<!-- Decorative image (empty alt) -->
<img src="decorative-swirl.png" alt="">

<!-- Image as link -->
<a href="/home">
  <img src="logo.png" alt="Company Name - Home">
</a>

Video Accessibility

  • Captions: Synchronized text for dialogue and sounds
  • Transcripts: Full text version of audio content
  • Audio descriptions: Narration of visual information

Keyboard Navigation

Many users can't use a mouse. Ensure your site works with keyboard alone:

Focus Management

/* Make focus visible */
:focus {
  outline: 3px solid #005fcc;
  outline-offset: 2px;
}

/* Don't remove focus outlines! */
/* BAD: :focus { outline: none; } */

Tab Order

Interactive elements should be focusable in a logical order. Avoid using tabindex greater than 0.

Skip Links

Provide a "skip to main content" link at the top of the page:

<a href="#main-content" class="skip-link">
  Skip to main content
</a>

<!-- Later in the page -->
<main id="main-content">...</main>

Color and Contrast

Contrast Ratios

WCAG requires minimum contrast ratios:

  • 4.5:1 for normal text (AA)
  • 3:1 for large text (18pt+ or 14pt bold)
  • 7:1 for enhanced contrast (AAA)

Don't Rely on Color Alone

Use additional indicators like icons, patterns, or text:

<!-- Bad: Color only -->
<span style="color: red">Error</span>

<!-- Good: Color + icon + text -->
<span class="error">
  <svg aria-hidden="true">...</svg>
  Error: Please enter a valid email
</span>

Forms

Labels

Every form input needs an associated label:

<!-- Method 1: Explicit association -->
<label for="email">Email address</label>
<input type="email" id="email" name="email">

<!-- Method 2: Implicit association -->
<label>
  Email address
  <input type="email" name="email">
</label>

Error Messages

Make errors clear and associate them with fields:

<label for="email">Email</label>
<input
  type="email"
  id="email"
  aria-describedby="email-error"
  aria-invalid="true"
>
<span id="email-error" class="error">
  Please enter a valid email address
</span>

Required Fields

Use the required attribute and indicate visually:

<label for="name">
  Name <span aria-label="required">*</span>
</label>
<input type="text" id="name" required>

ARIA: When HTML Isn't Enough

ARIA (Accessible Rich Internet Applications) adds accessibility info to custom components. But remember: No ARIA is better than bad ARIA.

Common ARIA Attributes

<!-- Labeling -->
<button aria-label="Close menu">X</button>

<!-- Describing -->
<input aria-describedby="hint">
<p id="hint">Password must be 8+ characters</p>

<!-- State -->
<button aria-expanded="false">Menu</button>

<!-- Live regions (for dynamic content) -->
<div aria-live="polite">Item added to cart</div>

Responsive and Mobile-First

Viewport Meta Tag

<meta name="viewport"
  content="width=device-width, initial-scale=1.0">

Mobile-First CSS

Start with mobile styles, then add complexity for larger screens:

/* Mobile first (default) */
.container {
  padding: 1rem;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
    max-width: 720px;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .container {
    max-width: 960px;
  }
}

Touch Targets

Make interactive elements at least 44x44 pixels with adequate spacing.

Testing Your Site

Automated Tools

  • WAVE: Browser extension for accessibility evaluation
  • axe DevTools: Comprehensive accessibility testing
  • Lighthouse: Built into Chrome DevTools
  • W3C Validator: Check HTML validity

Manual Testing

  • Keyboard only: Navigate using Tab, Enter, Space, Arrow keys
  • Screen reader: Test with NVDA (Windows), VoiceOver (Mac), or JAWS
  • Zoom: Test at 200% zoom
  • Color: View in grayscale to check color dependence

Checklist

  • All images have appropriate alt text
  • Color contrast meets WCAG AA (4.5:1)
  • All functionality available via keyboard
  • Focus indicators are visible
  • Form fields have associated labels
  • Page has proper heading structure
  • Skip link present and functional
  • No content flashes more than 3 times per second
  • HTML validates without errors
  • Site works on mobile devices
