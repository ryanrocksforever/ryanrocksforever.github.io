What is Universal Design?

Design that works for everyone, from the start.

Definition

Universal Design is the design of products, environments, and services to be usable by all people, to the greatest extent possible, without the need for adaptation or specialized design.

Unlike accessibility retrofitting, universal design is proactiveâ€”it considers the full range of human diversity from the very beginning of the design process.

The 7 Principles of Universal Design

Developed by a working group at North Carolina State University in 1997, these principles guide inclusive design:

1. Equitable Use

The design is useful and marketable to people with diverse abilities.

Web example: Providing both video and text versions of content.

2. Flexibility in Use

The design accommodates a wide range of individual preferences and abilities.

Web example: Allowing users to customize font size, color contrast, or input methods.

3. Simple and Intuitive Use

Use of the design is easy to understand, regardless of experience, knowledge, or language skills.

Web example: Clear navigation labels and consistent UI patterns.

4. Perceptible Information

The design communicates necessary information effectively, regardless of ambient conditions or sensory abilities.

Web example: Using icons with text labels, not just icons alone.

5. Tolerance for Error

The design minimizes hazards and adverse consequences of accidental or unintended actions.

Web example: Confirmation dialogs before destructive actions, undo functionality.

6. Low Physical Effort

The design can be used efficiently and comfortably with minimum fatigue.

Web example: Keyboard shortcuts, reducing the number of clicks needed.

7. Size and Space for Approach and Use

Appropriate size and space is provided regardless of user's body size, posture, or mobility.

Web example: Touch targets at least 44x44 pixels, adequate spacing between clickable elements.

WCAG: The Web Standard

The Web Content Accessibility Guidelines (WCAG) are the international standard for web accessibility. They're organized around four principles, remembered by the acronym POUR:

Perceivable

Information must be presentable in ways all users can perceive. This includes providing text alternatives for images, captions for videos, and sufficient color contrast.

Operable

Interface components must be operable by all users. This means keyboard accessibility, enough time to read content, and no content that causes seizures.

Understandable

Information and operation must be understandable. This includes readable text, predictable navigation, and input assistance for forms.

Robust

Content must be robust enough to be interpreted by various user agents, including assistive technologies. This means using valid, semantic HTML.

WCAG has three conformance levels:

  • Level A: Minimum accessibility (must have)
  • Level AA: Addresses major barriers (industry standard)
  • Level AAA: Highest level of accessibility (aspirational)

Types of Disabilities to Consider

When designing for universal access, consider users with:

Visual

  • Blindness
  • Low vision
  • Color blindness

Auditory

  • Deafness
  • Hard of hearing

Motor

  • Limited fine motor control
  • Slow response time
  • Limited mobility

Cognitive

  • Learning disabilities
  • Distractibility
  • Memory limitations

Universal Design vs. Accessibility

Aspect Accessibility Universal Design
Approach Reactive (often retrofitting) Proactive (built-in from start)
Focus People with disabilities All users across the spectrum
Goal Meet minimum requirements Optimize experience for everyone
Outcome Compliance Inclusion by design

Think of accessibility as the baseline and universal design as the aspiration.
